# Deploying this site to GitHub Pages

1. Replace the placeholder logo at `assets/logo.svg` with your real logo (recommended filename `assets/logo.png` or `assets/logo.svg`). The page currently references `assets/logo.svg`.
2. Commit and push the repository to GitHub (e.g., `main` branch).
3. In the repository settings on GitHub, enable GitHub Pages and set the source to the `main` branch (or `docs/` folder if you prefer).
4. After a minute or two, your site will be available at `https://<your-username>.github.io/<repo-name>/`.

Notes:

- The page uses a placeholder email `info@tanteta.com` — replace that with your actual contact address if desired.
- If you want a custom domain, add a `CNAME` file containing the domain and configure DNS accordingly.
